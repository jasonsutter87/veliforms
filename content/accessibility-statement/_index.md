# Accessibility Statement

**Last updated: December 13, 2025**

VeilForms is committed to ensuring digital accessibility for all users, including those with disabilities. We strive to meet the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards.

## Our Commitment

We believe privacy and security should be accessible to everyone, regardless of ability. That's why we're actively working to:

- Make VeilForms perceivable, operable, understandable, and robust
- Follow web accessibility best practices
- Test with assistive technologies
- Listen to user feedback and continuously improve

## Conformance Status

**Target:** WCAG 2.1 Level AA conformance

**Current status:** Partial conformance. We're actively addressing gaps and will update this statement as we improve.

### What This Means
- Some accessibility features are fully implemented
- Some areas need improvement (detailed below)
- We're committed to full conformance by Q2 2025

## Accessibility Features

### Currently Implemented

#### Keyboard Navigation
- All interactive elements are keyboard accessible
- Logical tab order throughout the interface
- Skip navigation links on main pages
- Focus indicators visible on all interactive elements
- No keyboard traps

#### Screen Reader Support
- Semantic HTML structure (headings, landmarks, lists)
- ARIA labels for complex interactions
- Alt text for informative images
- Decorative images marked with empty alt attributes
- Form fields properly labeled

#### Visual Design
- Minimum 4.5:1 contrast ratio for normal text
- Minimum 3:1 contrast ratio for large text and UI components
- Text resizable up to 200% without loss of functionality
- No information conveyed by color alone
- Responsive design for different viewport sizes

#### Forms and Interactions
- Clear error messages with suggestions
- Labels associated with form controls
- Required fields clearly marked
- Focus management in modals and dialogs
- Sufficient time limits (or ability to extend)

#### Content Structure
- Descriptive page titles
- Logical heading hierarchy (H1, H2, H3, etc.)
- Landmarks for navigation (header, main, footer, nav)
- Lists properly marked up
- Tables used for tabular data (with proper headers)

## Known Limitations

We're aware of the following accessibility issues and are working to fix them:

### Form Builder
- **Drag-and-drop:** Current implementation requires mouse. Planning keyboard alternative by Q1 2025.
- **Live preview:** Screen reader support needs improvement for real-time updates.
- **Field reordering:** Keyboard-only reordering not yet implemented.

### Dashboard
- **Charts/graphs:** Need better text alternatives and data tables.
- **Date pickers:** Some date selection interfaces require mouse interaction.
- **Tooltips:** Not all tooltips are accessible to keyboard/screen reader users.

### Documentation
- **Code examples:** Need better syntax highlighting contrast.
- **Diagrams:** Some technical diagrams lack detailed descriptions.

### Third-Party Components
- **Stripe payment widget:** Limited control over accessibility (working with Stripe on improvements).

## Assistive Technologies

VeilForms is tested with:

- **Screen readers:** NVDA (Windows), JAWS (Windows), VoiceOver (macOS/iOS), TalkBack (Android)
- **Browser zoom:** Up to 200% in Chrome, Firefox, Safari, Edge
- **Keyboard navigation:** All modern browsers
- **Voice control:** Dragon NaturallySpeaking, Voice Control (macOS)

If you use other assistive technologies, please let us know your experience.

## Browser and Device Compatibility

VeilForms aims to be accessible on:

- **Desktop browsers:** Chrome, Firefox, Safari, Edge (latest 2 versions)
- **Mobile browsers:** iOS Safari, Android Chrome
- **Operating systems:** Windows, macOS, iOS, Android, Linux

Older browsers may have limited accessibility support.

## Accessibility Testing

We conduct regular testing using:

- Automated tools (axe DevTools, Lighthouse, WAVE)
- Manual keyboard navigation testing
- Screen reader testing (NVDA, VoiceOver)
- Color contrast analyzers
- User testing with people who rely on assistive technologies

## Quick Accessibility Tips

If you're having trouble using VeilForms:

1. **Zoom in:** Use Ctrl/Cmd + Plus to increase text size
2. **Keyboard navigation:** Use Tab to move between elements, Enter to activate
3. **Skip links:** Press Tab on page load to reveal "Skip to main content" link
4. **Screen reader:** Landmarks and headings help navigate efficiently
5. **Need help?** Email accessibility@veilforms.com
